# FormulAI

**Transform Survey Feedback into Strategic Insights with AI**

### 2. Advanced AI Analytics

#### Automatic Theme Discovery
- **In-Vivo Coding**: Extract themes using respondents' exact words (not pre-defined categories)

- **Rare Insights**: Surface valuable minority perspectives (2-5% frequency)

#### Sentiment Intelligence
- **Multi-Level Analysis**: Overall sentiment + per-question sentiment
- **Emotional Tone Detection**: Beyond positive/negative - understand specific emotions
- **Climate Scoring**: Composite positivity score combining sentiment distribution
- **Sentiment Tracking**: See how sentiment correlates with specific topics

#### Correlation Discovery
- **Topic Co-occurrence**: Which themes appear together in responses?
- **Topic-Sentiment Links**: Which topics drive positive vs. negative sentiment?
- **Answer Segmentation**: How do closed-question answers relate to open-ended themes?
  - Example: "Premium users discuss 'advanced features' 3x more than free users"

#### Evidence-Based Insights
- **Key Findings**: AI-generated insights with confidence scores (high/medium/low)

## üî¨ Research Methodology

FormulAI applies proven qualitative research methods powered by AI:

### Grounded Theory
Build insights from the data itself rather than testing pre-existing hypotheses. Themes emerge from actual responses.

### In-Vivo Coding
Extract topics using respondents' exact language, preserving authentic voice and meaning.

### Theoretical Sampling
Intelligently select representative quotes that capture the essence of each theme while maintaining diversity.

### Sentiment Contextualization
Understand not just *what* people say, but *how they feel* about specific topics.

### Correlation Analysis
Discover hidden relationships: topic co-occurrence, topic-sentiment links, answer-topic segmentation.

## üöÄ Getting Started

See [TECHNICAL_STACK.md](./TECHNICAL_STACK.md) for complete installation and deployment instructions.

### Quick Start (5 minutes)
1. Clone repository
2. Start MongoDB with Docker: `npm run docker:up`
3. Install dependencies: `pnpm install`
4. Start dev servers: `pnpm run dev`
5. Visit `http://localhost:3000`

### Environment Setup
Configure OpenAI API key (or use local Ollama):
```env
OPENAI_API_KEY=your-key-here
```

### Create Your First Analytics Report
1. Create a form with open-ended questions
2. Collect at least 10 responses (tip: use test data for demo)
3. Click "Generate Analytics"
4. Watch real-time progress as AI analyzes responses
5. Explore insights through interactive dashboard

## üîß Configuration

### AI Provider Options

**OpenAI (Recommended)**:
```env
OPENAI_API_KEY=sk-...
```

**Ollama (Local/Private)**:
```env
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama3.1:8b
```

### Analytics Tuning

Adjust processing parameters in `server/src/analytics/utils/batch.processor.ts`:
- Chunk size: 10-25 responses per batch (adaptive based on length)
- Concurrency: 4 parallel chunks
- Target topics: 8-15 canonical themes

## üêõ Troubleshooting

**Analytics generation fails**:
- Verify OpenAI API key is valid
- Check MongoDB connection
- Ensure minimum 10 responses
- Review server logs for errors

**Slow processing**:
- Reduce chunk size in batch.processor.ts
- Use local Ollama for faster processing
- Check network latency to OpenAI API

**Memory issues**:
- Increase server memory allocation
- Process responses in smaller batches
- Limit concurrent analytics tasks
